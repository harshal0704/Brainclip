Complete system — four independent layers
Final architecture: Vercel (UI + orchestration) · Google Colab (voice only, BYOK GPU) · AWS Lambda (Remotion render, ~$0.03/video) · AWS S3 (per-user isolated storage). Each layer is independently replaceable.
Layer 1 Vercel
Next.js App Router. Auth, dashboard, job creation, Remotion Player preview, script editor, polling, delivery. Zero compute cost — serverless.
Layer 2 Colab
Fish Speech S2-Pro TTS only. FastAPI server exposed via Cloudflare Tunnel. Generates WAVs + word timestamps, uploads to user S3. Optional — fish.audio API replaces it.
Layer 3 Lambda
Remotion renderMediaOnLambda. Parallel frame rendering across ~50 Lambda functions. 2-min 720p video renders in ~30 sec. Costs ~$0.03–0.05/render.
Layer 4 S3
Per-user private bucket svgen-{userId}. Audio WAVs, transcript JSON, final MP4, voice refs, custom assets. Lifecycle: 30d audio, 7d video auto-expiry.
Data flow — one complete job lifecycle
1
Login → provision
Google OAuth via NextAuth. First login: create UUID, provision S3 bucket svgen-{userId} with private ACL + lifecycle rules. Store in DB. Subsequent logins skip.
2
Job creation → script gen
User picks duo, topic, style, background, voice mode. Next.js calls user's BYOK LLM key with the full system prompt → validates ScriptLine[] JSON → stores job in DB with status pending.
3
Presigned URL generation
Server generates presigned S3 PUT URLs for each expected file: audio/{jobId}/line_{n}.wav, transcripts/{jobId}/words.json, videos/{jobId}/final.mp4. Included in VoiceJobRequest.
4
Voice stage (Colab or fish.audio API)
Colab: POST /voice/job → Fish S2-Pro generates all speaker WAVs + faster-whisper word timestamps → uploads directly to user S3 via presigned PUTs → sets status voice_done. Fish.audio API mode: Next.js proxies TTS calls, uploads to S3 itself.
5
Render trigger
Next.js detects voice_done via polling → calls renderMediaOnLambda() with full ReelComposition inputProps including presigned GET URLs for audio + transcript. Stores renderId + bucketName in DB.
6
Lambda render
~50 Lambda functions render in parallel at 12 frames each. Remotion stitches chunks. Final MP4 uploaded to user S3 at videos/{jobId}/final.mp4. Lambda self-cleans after 7 days.
7
Delivery
Server generates 1-hour presigned GET URL for final video. Frontend shows Remotion Player preview + download button. Optional: copy to user Google Drive.
Resolution strategy
720p default 720×1280 vertical
Primary render quality. 30fps H.264. CRF 23. Render time ~25–35 sec on Lambda. Recommended for all final exports.
480p draft 480×854 vertical
Fast preview mode. CRF 26. Render time ~12–18 sec. Use for browser-preview before committing final 720p render. Saves Lambda cost during iteration.

Post-render actions
9

Final video plays in Remotion Player. Download button generates 1-hour presigned GET URL. Drive export button sends video to user's connected Drive folder.

Iterate
Edit any script line → re-TTS that line only (delta re-render) → re-render only the affected frames. Or change subtitle style/sticker → re-render video only, skip voice stage entirely (audio is cached in S3).

Same-region transfer free: Lambda in ap-south-1 reads S3 in ap-south-1 → $0 egress.
Background clips on R2: Cloudflare R2 has free egress. Host background video packs there, not S3.
IAM instance role: Lambda function role has s3:GetObject + s3:PutObject on svgen-* only. No long-lived keys.

Colab FastAPI
Fish Speech 1.5 (S2-Pro family) on Colab
Model: fishaudio/fish-speech-1.5 from HuggingFace (~1.5 GB). Download once per session, cache at /content/models/fish-speech/. Requires ~4GB VRAM → fits T4 free Colab GPU.
Zero-shot cloning: Pass 5–15 sec reference audio + transcript text. Model encodes speaker timbre as prompt_tokens. No fine-tuning. Cache tokens per speaker for the whole job session.
Batch inference: Send all lines for one speaker in a single call with the same prompt_tokens. ~3× faster than one call per line. Essential for staying within Colab session time.
Output: 44.1 kHz WAV per line. Concatenate with pause_ms silence gaps into master.wav. Upload per-line WAVs + master to S3 via presigned PUTs.
faster-whisper: Run on master.wav after TTS → word-level {word, start, end} timestamps → save as words.json to S3. Use large-v2 model for best accuracy.
from fish_speech.inference import InferenceEngine import soundfile as sf, io, boto3 engine = InferenceEngine("/content/models/fish-speech") # Encode reference once per speaker (cache per session) ref_tokens = engine.encode_reference( audio_path="/tmp/ref_a.wav", text="reference transcript text here" ) lines_a = [l for l in script_lines if l["speaker"] == "A"] for line in lines_a: wav_bytes = engine.tts( text=line["text"], prompt_tokens=ref_tokens, emotion=line["emotion"], # e.g. "excited" speaking_rate=line["speaking_rate"], temperature=line["temperature"], top_p=0.7, chunk_length=200, normalize=True ) # Upload to user S3 via presigned PUT requests.put(presigned_urls[line["id"]], data=wav_bytes)
Fish.audio hosted API — no Colab needed
When to use: No Colab session available, or user wants instant TTS without GPU cold-start (~5–10 min Colab warmup). User provides BYOK fish_api_key in settings.
Voice model registration: User uploads a ref clip once → POST /model → get persistent model_id. Stored in user profile per speaker. Reused for all future jobs.
TTS call: POST https://api.fish.audio/v1/tts with model_id, text, and style params. Returns streaming audio bytes.
Transcript generation: Next.js server uploads audio to S3, then calls faster-whisper running on a small EC2 t2.micro (or a free Colab notebook) to generate word timestamps. This is the only Colab dependency in API mode.
// Next.js API route — /api/tts/fish const ttsRes = await fetch("https://api.fish.audio/v1/tts", { method: "POST", headers: { Authorization: `Bearer ${user.fishApiKey}`, "Content-Type": "application/json" }, body: JSON.stringify({ model_id: speaker.fishModelId, text: line.text, format: "wav", latency: "normal", // "normal" | "balanced" chunk_length: 200, normalize: true, // emotion via system prompt prefix — fish.audio API // supports [emotion] tags inline in text }) }); const audioBuffer = await ttsRes.arrayBuffer(); await s3.putObject({ Bucket: userBucket, Key: `audio/${jobId}/${line.id}.wav`, Body: Buffer.from(audioBuffer) });
Custom reference clip cloning
Upload UX: Settings screen lets user upload 1–3 WAV/MP3 clips per speaker (5–15 sec each). App stores at voice-refs/{speakerId}/ref_{n}.wav in user S3.
Quality tips shown in modal: "Quiet room, no music, no reverb. Read at least 10 seconds of natural speech. Multiple clips improve consistency."
Multi-clip fusion: Colab averages the prompt_tokens across 2–3 clips before encoding. Better voice consistency, especially for longer scripts. Cached as speaker_tokens.pkl in S3.
Fish.audio API mode: Upload ref clip to POST /model → store returned model_id in user profile. S2-Pro on fish.audio uses it server-side.
Preset voice pack registry
Registry format: JSON metadata file in the app repo. Each entry has display name, persona, preview MP3 URL, language, gender, tags, colabRefUrl (reference clip), fishModelId (for API mode), recommendedEmotion, recommendedRate.
Colab mode: Colab fetches ref clip from colabRefUrl once, encodes prompt_tokens, caches as /content/cache/{packId}_tokens.pkl for the session. Repeat jobs using the same pack skip re-encoding.
API mode: Registry stores a fishModelId. App uses it directly — no download needed. Zero Colab dependency for preset voices in API mode.
Preview: UI plays 3-sec preview clip on hover using an HTML audio tag. No API call, no cost, instant feedback.
Fish Audio prosody + emotion parameters — full reference
emotion: neutral · happy · sad · angry · surprised · excited · whispering · shouting. Set per ScriptLine by the LLM based on dialogue context.
speaking_rate: 0.75–1.25. Slow dramatic = 0.8. Normal = 1.0. Fast energetic = 1.2. Per-line override in script JSON.
pause_ms: 150–800ms silence inserted between lines. Questions: 400–600ms. Punchlines: 600–800ms. Standard turn: 200ms.
temperature: 0.5 (stable) → 0.7 (natural) → 1.0 (expressive). Per-speaker setting, not per-line.
top_p: 0.6–0.9. Lower = more deterministic, better for consistent character voices. Use 0.7 as default.
chunk_length: 100–300 chars per synthesis chunk. 200 is optimal — longer gives more natural prosody across sentences.
normalize: Auto text normalisation for numbers, abbreviations, punctuation. Always true.
language: Auto-detect or explicit — en, zh, ja, ko, hi, fr, de, es, ar. S2-Pro handles code-switching (e.g. Hinglish).
Colab FastAPI server — full spec
Startup: Notebook cell installs fish-speech, faster-whisper, fastapi, uvicorn, cloudflared. Starts FastAPI on port 8000. Runs cloudflare tunnel → prints public URL. User pastes URL into app settings.
Session persistence: Models stay loaded in memory across jobs. Speaker token cache lives in /content/cache/. Background video packs cached in /content/bgcache/. All cleared on session end.
Health check: GET /health returns {status, models_loaded, gpu_memory_free, session_id}. App checks this before dispatching every job.
GET /health → {status, models_loaded, gpu_mb_free} POST /voice/job → accept VoiceJobRequest, return {jobId} immediately GET /voice/job/{jobId} → {stage, progressPct, error?} POST /voice/encode-ref → encode reference clip, return cache_key DELETE /voice/cache/{key} → clear cached speaker tokens
Complete LLM system prompt — paste-ready
Variables in {{double braces}} are injected from user settings and duo preset before sending. The LLM outputs only a JSON array — no markdown, no explanation.
You are a short-form video script writer specialising in engaging duo dialogue reels. Generate a natural conversation between Speaker A and Speaker B on the given topic. Output ONLY a valid JSON array of ScriptLine objects — no markdown, no code fences, no preamble, no explanation. === SPEAKER PERSONAS === Speaker A: {{speaker_a_persona}} Speaker B: {{speaker_b_persona}} Tone: {{tone}} (educational | debate | storytelling | comedy | interview) Language: {{language}} (en | hi | hinglish | zh | es | fr | auto) === FISH AUDIO PARAMETER GUIDE === Each object in the array must have EXACTLY these fields: "id": "line_001" through "line_020" (zero-padded) "speaker": "A" or "B" "text": the spoken line, 8–25 words, natural speech contractions "emotion": one of exactly: neutral, happy, sad, angry, surprised, excited, whispering, shouting → Match to dialogue content. Revelation moments = excited. Awe = whispering. → Do NOT overuse excited. Vary across the script. → Questions from an uncertain speaker = surprised. Confident claims = happy or neutral. "speaking_rate": float 0.75–1.25 → Fast energetic explanation = 1.1–1.2. Slow dramatic = 0.8. Normal = 1.0. → Last 2 lines must be 0.82–0.88 for impactful ending. "pause_ms": integer 150–800 → After a question: 400–600. Punchline before a reveal: 600–800. Standard: 200–300. → Use "..." in text for mid-sentence dramatic pauses; the TTS model handles them. "temperature": {{temp_a}} for A lines, {{temp_b}} for B lines (do not change these) "chunk_length": 200 (always exactly 200, do not change) "normalize": true (always exactly true, do not change) === WRITING RULES === 1. Total lines: 14–20. Alternate A and B. No speaker has 3+ consecutive lines. 2. Line 1: HOOK — surprising fact, bold claim, or "Did you know…" question. 3. Each line ≤25 words. Split longer thoughts across 2 lines. 4. Natural speech: contractions (it's, don't, you're), rhetorical questions, fillers (well, actually, wait—). 5. Speaker A uses "whispering" for awe moments. Speaker B uses "excited" for reveals. 6. Lines 1–3: fast pace (speaking_rate 1.05–1.15). Lines 4–12: varied. Last 2 lines: 0.82–0.88. 7. End with a memorable closing that invites the viewer to think or share. 8. NEVER include profanity, real names, copyrighted quotes, or medical/legal advice. 9. NEVER output anything outside the JSON array — not even a newline before [. === OUTPUT FORMAT === [ {"id":"line_001","speaker":"A","text":"...","emotion":"excited","speaking_rate":1.1,"pause_ms":250,"temperature":{{temp_a}},"chunk_length":200,"normalize":true}, {"id":"line_002","speaker":"B","text":"...","emotion":"surprised","speaking_rate":1.0,"pause_ms":350,"temperature":{{temp_b}},"chunk_length":200,"normalize":true} ]
Variables to inject
topic: user's input text speaker_a_persona: from selected duo preset speaker_b_persona: from selected duo preset tone: user-selected tone language: from user profile setting temp_a: voice pack A recommended temp (default 0.7) temp_b: voice pack B recommended temp (default 0.7)
Post-processing (always apply)
Strip any text before [ or after ] before JSON.parse.
Validate every required field present. Fill defaults: emotion→neutral, rate→1.0, pause→250, temp→0.7.
Cap text at 30 words — auto-split lines that overflow.
Reject arrays with fewer than 8 lines — re-call LLM with temperature +0.1 and retry once.

Speaker sticker system
SpeakerSticker component — full implementation spec
Active/idle state: getActiveSpeaker(frame, fps, scriptLines) returns "A"|"B"|null by checking current frame against each line's startSec–endSec range.
Glow ring: Pulsing ring around sticker image when speaking. interpolate(frame % 20, [0,10,20], [0.6,1,0.6]) for opacity. Color matches speaker.color. Idle ring: opacity 0.
Name tag: spring({frame: frameSinceSpeechStart, fps:30, config:{damping:12}}) for scale/opacity. Slides up from below the sticker when speaker becomes active. Fades on handoff.
Sticker shapes: Circle (default, borderRadius:50%), Rounded-square (borderRadius:24%), Hexagon (CSS clip-path polygon). User picks in editor.
Animation modes
Bounce
spring() scale on each line start. interpolate to 1.08× then back to 1.0 over 8 frames. Classic podcast reel. Default.
Slide-in
translateX from ±120px to 0 on speech start. translateX back to ±120px on handoff. spring() damping 14.
Zoom pulse
Slow interpolate from 1.0 to 1.06 over 30 frames while speaking. Snaps back on idle. Dramatic feel.
Float
Math.sin(frame * 0.08) * 6 → y-axis oscillation while speaking. Zero movement when idle. Calm/gentle.
Shake
Random x offset ±4px for first 6 frames on line start. High energy/comedy. Uses seeded random from line ID.
Static
No movement. Both stickers always visible. Only indicator is glow ring opacity. Clean minimalist look.
Overlay system
Reaction stickers (timed)
Placement: User taps a word in the script editor → picks an emoji → it appears at that word's timestamp in the video.
Animations: pop-fade (scale 0→1.4→1, then fade out over 45 frames), fly-in (translate from edge), bounce-out (scale down and disappear).
Remotion: Wrapped in <Sequence from={wordFrame} durationInFrames={45}>. Frame-accurate. Max 6 reaction stickers per video.
Structural overlays
Progress bar: Thin 4px bar at top. interpolate(frame, [0, durationInFrames], [0, 720]) → width in px. Color = speakerA.color.
Topic header: Slides down from top at frame 0, fades out at frame 75 (2.5 sec). Shows video topic text.
CTA end card: <Sequence from={durationInFrames - 60}> shows a styled overlay with user's CTA text. Fades in over 15 frames.

Complete edge case handling
Voice stage failures
Colab offline
Colab URL not reachable / tunnel expiredShow "Colab session expired" banner with retry button + link to notebook. Fallback offer: "Switch to Fish.audio API mode for this job." Store last-known Colab health timestamp; warn user if >2h since last ping.
Model OOM
Fish S2-Pro runs out of VRAM on Colab T4Colab returns {stage:"model_oom"}. App shows "GPU memory full — restart Colab runtime" with a link to the notebook. Suggest switching to fish-speech-1.4 (lighter, ~2.5 GB VRAM) in voice settings.
TTS partial fail
Some lines generated, others failed mid-jobColab saves successfully generated lines to S3 before the failure. App detects which lines are missing from S3. User can re-run TTS for failed lines only (delta re-TTS). No need to redo the full job.
Fish API error
fish.audio returns 401, 429, or 5xx401: "Invalid Fish.audio API key — update in settings." 429: exponential backoff 1s, 2s, 4s × 3 attempts, then "Fish.audio rate limit — try again in 60 sec." 5xx: retry 2× then "Fish.audio service unavailable — try Colab mode."
Bad ref clip
User uploads a noisy, short, or music-heavy reference clipValidate clip on upload: duration <3s → "Too short, 5+ seconds needed." SNR check via RMS analysis → warn if background noise detected. Clip duration >30s → auto-trim to first 15s. Show audio waveform preview so user can verify before saving.
Render stage failures
Lambda timeout
Render exceeds 120 sec Lambda timeoutThis shouldn't happen for ≤3 min videos with framesPerLambda:12, but set maxRetries:2 in renderMediaOnLambda. On timeout: show "Render timed out — retry at 480p draft first to test composition, then re-render at 720p."
Lambda not deployed
REMOTION_FUNCTION_NAME not found in AWS regionGET /api/render/health returns {status:"function_not_found"}. Dashboard shows "Remotion Lambda not set up" with a "Run setup" button that triggers the deploy script via a Next.js server action and shows live output.
Asset unreachable
Background clip URL returns 403/404 during renderRemotion's Lambda will fail to load the asset. Wrap renderMediaOnLambda in try/catch. Parse the error for "Failed to fetch" with the URL. Return specific error: "Background clip unreachable — choose a different clip or upload your own." Default to a solid colour background fallback if no clip available.
S3 upload fail
Lambda can't PUT to user S3 bucketIAM role check: Lambda execution role must have s3:PutObject on svgen-*. Return "Render complete but upload failed — check IAM permissions." Offer manual download from Lambda temp storage (Remotion stores render output in its own bucket temporarily).
LLM + script failures
Invalid JSON
LLM returns markdown-wrapped or truncated JSONStrip everything before [ and after ] before parsing. On parse failure: retry once with "Your previous response was not valid JSON. Output ONLY the raw JSON array." prepended. After 2 failures: show "Script generation failed — try a shorter topic or different LLM model."
Bad LLM key
LLM API returns 401 or 403Immediately surface "Invalid LLM API key" in the job form — validate the key on settings save, not at job time, by sending a cheap 5-token test prompt.
Short output
LLM returns fewer than 8 linesPost-processing rejects it. Retry once with temperature +0.1. If still short: show a warning "Script is shorter than ideal — continue anyway or regenerate."
S3 + auth edge cases
Bucket exists
S3 bucket name already taken on first login (UUID collision)Extremely unlikely but handle: catch BucketAlreadyExists on createBucket, generate a new UUID, retry once. After 2 failures, use user UUID + timestamp suffix.
Presigned expiry
User sits on the editor for >15 min before triggering renderPresigned PUT URLs expire after 15 min. On job dispatch, check age of job record. If >12 min old: silently regenerate new presigned URLs before dispatching to Colab. User never sees the error.
Video expired
User tries to download a video after the 7-day S3 expiryServer checks if s3Key still exists before generating a presigned GET URL. If not: show "This video has expired. Re-render to generate a new copy." Job script and voice refs are still in S3 (30-day expiry) — offer one-click re-render skipping voice stage.
Drive auth
Google Drive OAuth token expiredCatch 401 on Drive upload. Redirect user to re-authorize Drive scope. Offer to keep the S3 video available for 24 more hours as a backup while they re-auth.
